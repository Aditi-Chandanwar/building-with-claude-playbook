# Lesson 02 - Overview of Claude Models

---

# 🎯 Learning Objective

Understand the strengths, trade-offs, and ideal use cases of Claude's three model families so the right model can be selected for different business and engineering problems.

---

# 🧠 Core Concept

Not every AI task requires the most powerful model.

Claude provides three model families that balance **intelligence, speed, and cost**. Choosing the appropriate model improves application performance while reducing operational costs.

---

# 🔑 Key Takeaways

- Claude Opus is the most capable model for complex reasoning tasks.
- Claude Sonnet provides the best balance of intelligence, speed, and cost.
- Claude Haiku is optimized for low latency and high-volume applications.
- Model selection should be based on task requirements rather than always choosing the most powerful model.

---

# 📊 Claude Model Comparison

| Model  | Intelligence | Speed      | Cost   | Best For                                                |
| ------ | ------------ | ---------- | ------ | ------------------------------------------------------- |

| Haiku  | ⭐⭐⭐       | ⭐⭐⭐⭐⭐ | Low    | Fast responses, chatbots, classification, summarization |

---

# 💡 Why This Matters

Using a larger model for every task increases cost and response time.

A good AI engineer chooses the smallest model that can reliably complete the task.

This leads to:

- Lower API costs
- Faster applications
- Better scalability

---

# 🏗 Engineering Principle

> Choose the smallest model capable of solving the problem reliably.

Bigger models are not always better.

Good engineering optimizes for quality, speed, and cost.

---

# ⚠ Common Mistakes

- Using Opus for simple classification tasks.
- Selecting models based only on intelligence.
- Ignoring API costs when scaling applications.
